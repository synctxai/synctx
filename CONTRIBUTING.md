# Contributing to SyncTx

SyncTx is an early-stage protocol project. Contributions are welcome, especially when they improve contract safety, verifier reliability, agent workflows, examples, tests, or documentation.

## Good First Contributions

- Fix unclear documentation.
- Add focused tests for contracts, verifier services, or skills.
- Improve example deal contracts or verifier specs.
- Report confusing agent workflows with reproduction steps.
- Propose new deal contract patterns with clear settlement rules.

## Before Opening a Pull Request

1. Keep the change scoped. Avoid mixing contract changes, docs rewrites, and workflow changes in one PR.
2. Explain the motivation and risk in the PR description.
3. Add or update tests when behavior changes.
4. Do not include private keys, seeds, API tokens, deployment secrets, or real user data.
5. For contract changes, document any state-machine, timeout, fee, verifier, or settlement behavior change.

## Issues

Use GitHub Issues for bugs, proposals, documentation gaps, and example requests. Include reproduction steps when reporting a bug.

## Security

Do not open public issues for vulnerabilities that could affect funds, private keys, verifier integrity, or platform indexing. Contact the maintainers privately first.
